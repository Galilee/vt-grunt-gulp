1 - Grunt / Gulp - Présentation

  • Outils développés en Javascripts et basés sur NodeJS
  • Outils d'automatisation
  • On appelle ça des "Tasks Runners"
  • Exemples : pre-processing, minification, compilation, TU, linting etc.
  • Gestion de workflow au niveau des tasks
  • Beaucoup de plugins disponibles
  • Objectif : industrialiser et gagner du temps sur les tâches ingrates

1 - Grunt / Gulp - Pré-requis et installation

  • Nécessite NodeJS
  • Nécessite l'installation de NPM (gestionnaire de dépendances)
  • Installation de grunt-cli et de gulp-cli en une ligne de commande 
    • npm install grunt-cli -g
    • npm install gulp-cli -g
  • Toujours avoir un Gruntfile.js / Gulpfile.js là où on exécute notre commande

1 - Grunt / Gulp - Grunt VS Gulp (1/2)

  • Globalement, ils sont identiques hormis quelques éléments
  • Grunt
    • Avantages
      • Là depuis plus longtemps donc grosse communauté
      • Plus de plugins (~6k)
    • Inconvénients
      • Mauvaise gestion des streams (séquentielle, mauvaises performances, répétition obligatoire lors de modifications)
      • Workflow totalement synchrone (pas d'async possible)

1 - Grunt / Gulp - Grunt VS Gulp (2/2)

  • Gulp
    • Avantages
      • Utilisation des streams NodeJS (meilleur performances)
      • Workflow aynschrone si besoin
    • Inconvénients
      • Plus jeune que Grunt donc moins de plugins
    • Résumé
      • ”Gulp propose d'agir sur des flux de fichiers qui peuvent eux-mêmes proposer leur contenu sous forme de Buffer ou un flux de leur propres données.”
